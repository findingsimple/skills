# Vault Linker

Scan Obsidian vault files and add `[[wiki links]]` to known entities (people, incidents, triage issues). Idempotent — safe to re-run.

## Architecture

This skill uses **one Python script** (`link.py`) in the skill directory. It discovers linkable entities by walking the vault filesystem, then scans target files and replaces plain-text mentions with wiki links.

### Key principles

**Idempotent:** Already-linked text is skipped. Running multiple times produces the same result.
**Safe:** Never modifies code blocks or existing links. Adds wiki links to YAML frontmatter fields (`team:`, `participants:`, `person:`) and body content. Atomic file writes via `.tmp` + `os.replace()`.
**Scoped:** Can target specific file groups (retros, bonusly, incidents, triage) or all at once.

## Instructions

Follow these steps exactly.

### Step 0 — Parse arguments

Check `$ARGUMENTS` for flags:

- `--dry-run`: print proposed changes without modifying files
- `--scope <value>`: one of `all`, `retros`, `bonusly`, `incidents`, `triage` (default: `all`)

Examples:
- `/vault-linker --dry-run` — dry run, all scopes
- `/vault-linker --scope retros` — link retro files only
- `/vault-linker --dry-run --scope incidents` — dry run, incidents only

> **Note:** Sprint summary files are not linked — assignee names in sprint tables create noise in the graph view without adding navigational value. Person links in retros and bonusly entries are more meaningful. The sprint timeline index (`_Timeline.md`) is still generated when scope is `all`.

### Step 1 — Resolve vault path

Check that `OBSIDIAN_TEAMS_PATH` is set:
```bash
echo "$OBSIDIAN_TEAMS_PATH"
```

The **vault base** is the parent directory of the teams path (i.e., the HappyCo directory). If `OBSIDIAN_TEAMS_PATH` is not set, stop and tell the user.

Verify the path exists with `ls`.

### Step 2 — Run link.py

```bash
python3 ~/.claude/skills/vault-linker/link.py --vault-base "{vault_base}" --teams-path "{teams_path}" --scope "{scope}" [--dry-run]
```

The script:
1. Discovers linkable entities (people, incidents, triage issues) by walking the vault
2. Scans target files based on `--scope`
3. Replaces plain-text mentions with wiki links in body content (skipping code blocks, existing links)
4. Adds `person:` backlinks to feedback files (bonusly, review cycles)
5. Links `team:` and `participants:` YAML frontmatter fields to vault pages
6. Writes modified files atomically (or prints changes in dry-run mode)
7. Generates team hub pages, sprint timeline indexes, and plans index when scope is `all`
8. Prints a summary of changes made

> **Note:** The incident index (`Incidents/_Index.md`) is generated by `/incident-kb`, not this skill — it has access to severity data from the incident objects.

### Step 3 — Confirm completion

Show the user the summary output. If dry-run, ask if they want to proceed with the real run.
